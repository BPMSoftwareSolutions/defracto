# Reports Sketches

This folder holds run-level ASCII visual evidence for meaningful slices.

Each sketch is a governed proof surface, not decoration.

Current convention:

- The sketch must show what the run achieved.
- The sketch must distinguish passed, blocked, deferred, and stale artifacts.
- The sketch must link back to the textual slice report and any receipts or explicitly note why receipts are deferred.
- The sketch must not invent product truth.
- The sketch starts as ASCII and can later be rendered through the `visualProjectionModel`.
